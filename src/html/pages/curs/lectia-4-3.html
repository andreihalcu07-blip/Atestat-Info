Modul 4 — Arhitectură Hardware

Lecția 4.3 — RAM și Cache: Ierarhia Memoriei

De ce avem L1, L2, L3 și RAM? Viteză vs capacitate. Ierarhia e cheia performanței.

Începe Lecția

Introducere

Memorie ierarhică: Cu cât e mai rapidă, cu atât e mai mică și mai scumpă. Cu cât e mai mare, cu atât e mai lentă.

PS5: Cache L1 (32KB per core), L2 (512KB), L3 (8MB). RAM = 16 GB GDDR6 la 448 GB/s.

Scop: CPU să aibă date rapide aproape. Cache reduce latența de la 200 ns (RAM) la 1 ns (L1).

Teorie Structurată

4.1 — RAM (DRAM)

DRAM: Fiecare bit stocat într-un condensator mic + tranzistor. Se descarcă, deci necesita refresh la fiecare 64 ms.

Latency: 50-100 ns (100+ ciclu la 3.5 GHz). Complex. Dar capacitate mare.

GDDR6: VRAM high-speed. 16 GB PS5, 448 GB/s bandwidth.

4.2 — Cache L1, L2, L3

L1: 32 KB, 1-3 ns latență. Ultra rapid, stocat aproape de core.

L2: 512 KB, 4-10 ns. Mai lent dar încă rapid.

L3: 8 MB shared, 15-25 ns. Un cache pentru tot CPU.

SRAM: Cache = SRAM (flip-flop) — nu necesită refresh, dar ocupă mult silicon.

4.3 — Cache Hit & Miss

Hit: Data găsită în cache → latență mică.

Miss: Data absentă → se fetch-ează din memorie. Penalty 100+ ciclu.

Hit rate: CPU modern > 90%. În game loops, tipic 95-98% dacă data locală. Cache miss rate de 2% = degrade performance 20%.

4.4 — Localitate Temporală și Spațială

Temporal: Dacă accesezi data acum, probabil o vei accesa curând din nou. Cache reține.

Spațial: Dacă accesezi un address, probabil vei accesa și vecinii. Cache line = 64 bytes. Încarcă întreg bloc.

Exemplu: Array de 1000 ints. Acces secvențial = 1000/16 cache lines. Acces random = 1000 cache lines = 16× mai multe misses.

4.5 — Eviction și Prefetch

Eviction: Cache full. Trebuie să scoată o linie veche. Algoritm LRU (Least Recently Used).

Prefetch: CPU ghicește ce data vei folosi și o aduce înainte. Dacă ghicește bine = minus latență. Dacă greșește = ocupă cache inutil.

Preload in PS5: Game engine prefetch-ează asset-uri înainte de frame render (streaming textures).

Legătura Fizică – Informatică

Fizică: SRAM (cache) = flip-flop → rapid, scump. DRAM (RAM) = condensator → lent, ieftin.

Informatică: Algoritmi cache-friendly (localitate bună) → FPS mai mare.

Aplicare Directă

Exemplu: Traversare Array în Engine

Array 1,000,000 vectori (12 bytes each). Traversare secvențială = cache friendly.

Cache line 64 bytes. Încarcă 5 vectori per line. Total = 200,000 cache lines.

Acces random: 1,000,000 cache lines. 5× mai multe miss. FPS scade drastic.

Exemplu Real: PS5 Streaming Textures

Scenario: Open-world game. Player merge rapid. Engine trebuie să încarce textures ahead of time.

Cache usage: Textures sunt mari (MB). L3 cache mic (8 MB) → miss rate mare dacă nu prefetch.

Solution: Streaming system: anticipare camera movement. Încarcă textures înainte (prefetch). L3 cache și VRAM se umple cu data necesara. FPS constant.

Probleme Frecvente

⚠️ De ce cache miss rate mic produce impact mare?

Miss penalty = 200 ciclu. 1% miss rate × 200 = +2 ciclu mediu per instr. CPU 3.5 GHz, latenta ridicată. Perf -30%.

⚠️ De ce L1 cache este mic?

SRAM costă mult. 32 KB la fiecare core = ~2 milioane tranzistori. L1 mai mare ar crește latenta și consumul.

Recapitulare

  • RAM: DRAM mare, 50-100 ns latență. 16 GB GDDR6 PS5.
  • L1/L2/L3: SRAM rapid, mic. L1 ~1 ns, L3 ~15 ns.
  • Cache hit: Data găsită local → rapid. Miss → penalty 100+ cicli.
  • Localitate: Temporală & spațială. Access pattern defineste performance.
  • Prefetch: aduce data înainte. Dacă ghicește corect, reduce latență.
  • Game engine optimizează structuri de date pentru cache-friendly behavior.

Quiz — 5 Întrebări

Întrebarea 1

RAM (DRAM) stocheaza bitul prin:

  • a) Flip-flop
  • b) Condensator
  • c) Releu
  • d) Curent continuu stabil
Arată răspunsul

b) Condensator — Necesită refresh periodic.

Întrebarea 2

L1 cache latență aproximativ?

  • a) 1-3 ns
  • b) 10-20 ns
  • c) 50-100 ns
  • d) 1 ms
Arată răspunsul

a) 1-3 ns — L1 e cel mai rapid.

Întrebarea 3

Localitate spațială înseamnă:

  • a) Acces repetat aceeași adresă
  • b) Acces vecini consecutivi în memorie
  • c) Acces random
  • d) Acces la GPU
Arată răspunsul

b) Acces vecini consecutivi — Cache line profită.

Întrebarea 4

Cache miss rate 1% cu penalty 200 ciclu produce CPI extra:

  • a) 0.02 ciclu
  • b) 1 ciclu
  • c) 2 ciclu
  • d) 200 ciclu
Arată răspunsul

c) 2 ciclu — 0.01 × 200 = 2.

Întrebarea 5

Prefetch ajută când:

  • a) Acces random total
  • b) Există pattern predictibil de acces
  • c) Datele nu sunt folosite
  • d) Cache e infinita
Arată răspunsul

b) Pattern predictibil — Prefetch ghicește următoarele adrese.

Exercițiu Aplicat

🧠 Exercițiu: Cache Efficiency

Cerință: Vector 1,000,000 ints. Cache line 64 bytes (16 ints). Acces secvențial vs random.

  1. Cache lines necesare secvențial
  2. Cache lines necesare random
  3. Raport miss rate
  4. Impact pe perf dacă miss penalty 200 ciclu
Arată rezolvarea

1. Secvențial: 1,000,000 / 16 = 62,500 cache lines.

2. Random: 1,000,000 cache lines (1 per int).

3. Ratio: Random/Sequential = 16× misses.

4. Perf impact: Dacă secvențial: miss rate 6.25%. Random: 100% miss rate. CPI +200 ciclu per access. Perf drop ~16× (dramatic).
