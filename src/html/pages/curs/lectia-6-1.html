Modul 6 — Alimentare și Probleme Reale

Lecția 6.1 — Secvența de pornire (Power-On Sequence)

De la 5V standby la init APU. Fiecare rail are un moment precis.

Începe Lecția

Introducere

Power-on sequence: Ordinea în care rail-urile și subsistemele pornesc.

Dacă ordinea e greșită, CPU pornește fără memorie, VRM se oprește, sau consolea intră în protecție.

În PS5, secvența completă durează sub 1 secundă: Standby → Main → VRM → APU → Boot.

Teorie Structurată

6.1 — Starea Standby

5V standby: activ mereu. Alimentează controller power, IR receiver, USB charging.

Microcontroller: SoC mic care detectează butonul power sau semnal din controller.

Consum tipic: 1-3W.

6.2 — Semnal Power-On

Power button: Trimite semnal către PMIC (Power Management IC).

PS_ON: Semnal logic către PSU pentru a activa 12V main.

PSU răspunde cu 12V în 50-200 ms. Dacă nu, consola rămâne în standby.

6.3 — Secvența Rail-urilor

Ordine tipică: 12V main → 3.3V → 1.8V → 1.05V core → 1.35V GDDR6.

Power Good: Fiecare rail trimite PG când stabil. Dacă lipsesc PG, secvența se oprește.

Delay: 1-10 ms între rail-uri pentru stabilitate.

6.4 — Init Firmware

Boot ROM: CPU citește instrucțiuni din flash BIOS. Inițializează RAM, GPU, SSD.

Memory training: GDDR6 calibrat (timing, voltage). Dacă training fail, reboot sau no boot.

Device init: USB, WiFi, audio. Apoi OS boot.

6.5 — Protecții în Secvență

Watchdog: Dacă boot nu se finalizează în timp, se face reset.

OCP/OVP: Dacă curentul sau tensiunea depășește limite, VRM oprește rail-ul.

Consola poate blink LED sau beep, semn că secvența s-a oprit.

Legătura Fizică – Informatică

Fizică: Rail-uri sunt linii electrice cu tensiuni precise.

Informatică: Firmware pornește doar când rail-urile sunt stabile și PG este valid.

Aplicare Directă

Exemplu: Secvență PS5

1. 5V standby activ.

2. Buton power → PS_ON activ.

3. 12V main on. PG_12V OK.

4. 3.3V și 1.8V on. PG_3V3, PG_1V8 OK.

5. 1.05V core on. PG_CORE OK.

6. BIOS init, RAM training, OS boot.

Exemplu Real: Boot Loop

Consola pornește 1 sec, se oprește. Cauză: PG_CORE lipsește → VRM detectează scurt.

Firmware vede rail instabil și face reset. Rezultat: boot loop.

Probleme Frecvente

⚠️ PS_ON nu pornește PSU

Controller power defect sau semnal PS_ON absent. Consola rămâne în standby.

⚠️ Memory training fail

RAM GDDR6 defectă sau rail 1.35V instabil. Consola se oprește la 1-2 sec.

Recapitulare

  • Standby: 5V activ, controler power treaz.
  • PS_ON: Activează 12V main.
  • Rail-uri: Pornește în ordine cu PG valid.
  • Boot ROM: Inițializează RAM și GPU.
  • Protecții: OCP/OVP/Watchdog opresc secvența dacă ceva e greșit.

Quiz — 5 Întrebări

Întrebarea 1

Care rail este activ mereu?

  • a) 12V main
  • b) 5V standby
  • c) 1.05V core
  • d) 3.3V main
Arată răspunsul

b) 5V standby

Întrebarea 2

PS_ON face:

  • a) Oprește RAM
  • b) Activează 12V main
  • c) Oprește GPU
  • d) Reset BIOS
Arată răspunsul

b) Activează 12V main

Întrebarea 3

Power-good semnal indică:

  • a) Rail stabil
  • b) GPU overclock
  • c) RAM defect
  • d) PSU off
Arată răspunsul

a) Rail stabil

Întrebarea 4

Memory training se face la:

  • a) În standby
  • b) În boot ROM
  • c) După OS
  • d) Niciodată
Arată răspunsul

b) În boot ROM

Întrebarea 5

Dacă PG_CORE lipsește:

  • a) Boot continuă
  • b) Consola face reset/stop
  • c) GPU accelerează
  • d) SSD se oprește
Arată răspunsul

b) Consola se oprește

Exercițiu Aplicat

🧠 Exercițiu: Diagnostic Secvență

Cerință: Consola pornește 0.5 sec și se oprește. Măsurători: 5V standby OK, 12V OK, 3.3V OK, 1.8V OK, 1.05V = 0V.

  1. Unde e problema?
  2. Ce suspectezi: VRM, APU scurt, semnal enable?
  3. Ce următor pas?
Arată rezolvarea

1. Problema este rail-ul 1.05V core (APU).

2. Suspect VRM defect sau scurt pe APU rail. Verifică rezistența la masă.

3. Măsoară MOSFET-uri și bobine VRM, verifică enable signal. Dacă scurt, izolează condensatori sau APU.
