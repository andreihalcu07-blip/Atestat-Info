Modul 6 — Alimentare și Probleme Reale

Lecția 6.4 — Instabilitate

Restarturi, freeze, shutdown aleator. Cum corelezi simptome cu rail-uri și temperaturi.

Începe Lecția

Introducere

Instabilitate: Consola pornește, dar se resetează, se blochează sau se oprește fără motiv aparent.

De obicei cauza este termică, rail instabil sau defecte intermitente (BGA, VRM, RAM).

Instabilitatea este cea mai grea de diagnosticat, pentru că apare doar în anumite condiții.

Teorie Structurată

6.1 — Thermal Throttling vs Shutdown

Throttling: Firmware reduce frecvența când temperatura crește. FPS scade, dar consola rămâne on.

Shutdown: Dacă temperatura depășește pragul (90-100°C), sistemul se oprește pentru protecție.

Instabilitate termică apare după 5-20 min de joc.

6.2 — Rail Instabil și Ripple

Ripple mare: VRM degradează, condensatori uscați.

Symptom: Crash în jocuri heavy, dar meniu OK.

Rail core 1.05V trebuie să aibă ripple < 20 mV. Dacă 60 mV, APU devine instabil.

6.3 — RAM Errors

GDDR6 timing: Dacă voltaj 1.35V scade sau memory training este marginal, erorile apar la load mare.

Symptom: Artefacte grafice, crash după 1-2 ore.

Test: Memory stress test, observare temperatură RAM.

6.4 — BGA Microfracturi

BGA balls: APU și RAM sunt BGA. Dilatare termică repetată poate crea microfracturi.

Symptom: Consola funcționează când e rece, dar crash când se încălzește.

Diagnostic: Presiune ușoară pe APU poate schimba comportamentul (test de flex).

6.5 — PSU și Power Spikes

PSU slab: Nu poate livra spike-uri de curent. Rail 12V scade scurt → reset.

Symptom: Reset imediat când începe joc (load mare).

Test: Măsoară 12V cu osciloscop, urmărește droop la load.

Legătura Fizică – Informatică

Fizică: Temperatură și alimentare instabilă afectează tranzistorii.

Informatică: Erori de calcul duc la crash, freeze, reset.

Aplicare Directă

Exemplu: Restart în joc

Joc heavy → GPU load 95% → curent mare. VRM ripple crește, APU crash, consolă restart.

În meniu, load mic, nu apare problemă. Diagnosticul trebuie făcut sub load.

Exemplu Real: PS5 Freeze după 30 min

Consola funcționează 30 min, apoi freeze. Temperatura APU 92°C.

Curățare radiator, schimbare pastă termică. Temperatura scade la 75°C. Freeze dispare.

Probleme Frecvente

⚠️ Crash aleator după ore

RAM marginală sau BGA microfracturi. Diagnostic dificil, necesită test termic.

⚠️ Restart imediat sub load

PSU slab sau VRM insuficient. Rail 12V droop.

Recapitulare

  • Thermal: Throttling sau shutdown la 90-100°C.
  • Ripple: VRM instabil = crash sub load.
  • RAM errors: Artefacte și freeze.
  • BGA: Microfracturi termice dau instabilitate.
  • PSU: Power spikes pot cauza reset instant.

Quiz — 5 Întrebări

Întrebarea 1

Throttling este:

  • a) Oprire totală
  • b) Reducere frecvență pentru răcire
  • c) Creștere tensiune
  • d) Reset BIOS
Arată răspunsul

b) Reducere frecvență

Întrebarea 2

Ripple mare pe 1.05V core produce:

  • a) FPS mai mare
  • b) Crash și instabilitate
  • c) Temperatură scăzută
  • d) Boot rapid
Arată răspunsul

b) Crash

Întrebarea 3

Freeze după 30 min indică:

  • a) Standby error
  • b) Problemă termică sau BGA
  • c) BIOS corupt
  • d) SSD defect imediat
Arată răspunsul

b) Problemă termică

Întrebarea 4

Restart imediat sub load indică:

  • a) PSU droop
  • b) WiFi error
  • c) Update pending
  • d) LED defect
Arată răspunsul

a) PSU droop

Întrebarea 5

BGA microfracturi apar din:

  • a) Software
  • b) Dilatări termice repetate
  • c) Overclock GPU
  • d) RAM update
Arată răspunsul

b) Dilatări termice

Exercițiu Aplicat

🧠 Exercițiu: Diagnosticul Instabilității

Cerință: Consola face freeze după 20 min în joc. Temperatură APU 95°C, rail 1.05V ripple 25 mV.

  1. Care este cauza principală?
  2. Ce test faci primul?
  3. Ce soluție aplici?
Arată rezolvarea

1. Cauza principală: temperatură prea mare (95°C) + ripple ușor ridicat.

2. Test: curățare radiator și schimbare pastă termică. Monitorizează temp sub load.

3. Soluție: reducerea temperaturii, verificare ventilator. Dacă persistă, verifică VRM condensatori.
