Modul 5 — Anatomia unei Console

Lecția 5.2 — APU (CPU + GPU pe un singur cip)

De ce consolele folosesc APU? Latenta mică, cost redus, eficiență mare.

Începe Lecția

Introducere

APU (Accelerated Processing Unit): CPU și GPU integrate pe același die de siliciu.

Avantaj: comunicație internă rapidă, memorie comună (UMA), cost și consum reduse.

PS5 APU: 7 nm, ~308 mm² die, CPU Zen 2 + GPU RDNA pe același chip.

Teorie Structurată

5.1 — CPU și GPU pe același die

CPU: Control logic, AI, physics, game loop.

GPU: Rendering grafic și compute masiv.

Integrarea reduce distanță și latență: datele trec prin Infinity Fabric intern, nu prin PCIe extern.

5.2 — Memorie unificată (UMA)

UMA: CPU și GPU împart aceeași RAM (GDDR6). Elimină copierea între VRAM și RAM.

Beneficiu: Fără duplicare asset-uri. Texture, mesh, audio în același spațiu memorie.

Dezavantaj: CPU și GPU concurează pentru bandwidth. QoS și scheduling sunt critice.

5.3 — Cache Coherency

Problemă: CPU scrie date în cache. GPU citește. Dacă GPU citește înainte de writeback, vede date vechi.

Coherency: Protocol hardware (MESI) asigură că GPU vede ultima versiune. Necesită invalidare cache și semnale bus.

Cost: Coherency adaugă latență și overhead.

5.4 — Die Size și Yield

Die mare: APU combină CPU+GPU, deci die = mare. 308 mm² PS5. Mai mare = mai multe defecte per wafer.

Yield: Procent de chip-uri funcționale. Die mare reduce yield → cost ridicat.

Compromis: Console preferă APU pentru cost global și simplitate producție.

5.5 — Power și Thermal Budget

TDP total: CPU + GPU împart același buget termic. PS5 TDP ~200W pentru întreg APU + rest sistem.

Dynamic allocation: Dacă GPU lucrează intens, CPU reduce frecvența (DVFS). Alternativ, CPU high, GPU throttle.

APU necesită răcire eficientă, heat spreader mare și presiune constantă a radiatorului.

Legătura Fizică – Informatică

Fizică: APU = un singur chip masiv cu miliarde de tranzistori. Interconnect intern mai rapid decât PCB.

Informatică: UMA reduce overhead. Memory management e unificat, dar impune scheduling atent.

Aplicare Directă

Exemplu: Shared Buffer CPU-GPU

CPU calculează poziții pentru 100,000 obiecte și scrie într-un buffer. GPU citește același buffer pentru rendering.

În APU, buffer este în GDDR6 comun. Nu mai este nevoie de copy CPU→VRAM. Latenta scade cu 2-3 ms per frame.

Exemplu Real: PS5 APU Specs

PS5 APU: 8-core Zen 2 @ 3.5 GHz și 36 CU RDNA @ 2.23 GHz. Fabricat 7 nm.

Die size ~308 mm². Consum total APU ~160W în full load. VRM livrează 1.0-1.2V core, 90A+ curent.

APU și RAM pe același PCB scurtează trasee, reduce latență pentru memory bus.

Probleme Frecvente

⚠️ De ce APU este greu de reparat?

BGA cu mii de bile. Reballing este delicat. O singură bile defectă = no boot sau artifacting.

⚠️ Thermal throttling

Dacă APU depășește 95°C, firmware reduce frecvența. FPS scade. Cauze: praf, pastă termică uscată, ventilație slabă.

Recapitulare

  • APU: CPU + GPU pe același die.
  • UMA: Memorie comună. Fără copy CPU↔GPU.
  • Coherency: Protocol asigură date consistente între CPU și GPU.
  • Die mare: Cost mai mare, yield mai mic.
  • Thermal: CPU și GPU împart buget termic. DVFS indispensabil.

Quiz — 5 Întrebări

Întrebarea 1

APU înseamnă:

  • a) CPU separat de GPU
  • b) CPU și GPU pe același die
  • c) Doar GPU dedicat
  • d) Doar CPU dedicat
Arată răspunsul

b) CPU și GPU pe același die

Întrebarea 2

UMA permite:

  • a) Memorie separată
  • b) Copy CPU↔GPU obligatoriu
  • c) Memorie comună CPU+GPU
  • d) Fără cache
Arată răspunsul

c) Memorie comună

Întrebarea 3

Coherency protocol asigură:

  • a) Putere mai mare
  • b) Date consistente între CPU și GPU
  • c) FPS mai mare întotdeauna
  • d) Latență zero
Arată răspunsul

b) Date consistente

Întrebarea 4

Die mare →

  • a) Yield mai mic
  • b) Cost mai mic
  • c) Fără defecte
  • d) Mai puțin complex
Arată răspunsul

a) Yield mai mic

Întrebarea 5

PS5 APU fabricat la:

  • a) 14 nm
  • b) 10 nm
  • c) 7 nm
  • d) 5 nm
Arată răspunsul

c) 7 nm

Exercițiu Aplicat

🧠 Exercițiu: Bandwidth UMA

Cerință: CPU cere 40 GB/s, GPU cere 300 GB/s. RAM total 448 GB/s. Cât headroom rămâne?

  1. Calculează total demand
  2. Este saturat bus-ul?
  3. Ce headroom rămâne pentru I/O?
Arată rezolvarea

1. Total = 40 + 300 = 340 GB/s.

2. 340 < 448, deci nu e saturat.

3. Headroom = 448 - 340 = 108 GB/s pentru I/O, audio, streaming.
