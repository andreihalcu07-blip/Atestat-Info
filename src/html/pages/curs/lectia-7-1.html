Modul 7 — Performanță și Grafică

Lecția 7.1 — Pipeline grafic: de la vertex la pixel

Randare modernă: geometrie, shader-e, rasterizare, post-procesare.

Începe Lecția

Introducere

Pipeline grafic: Set de etape care transformă scene 3D în pixeli 2D.

Fiecare etapă are cost. Un bottleneck la o etapă poate reduce FPS.

PS5 GPU execută pipeline-ul pentru fiecare frame, 60 ori pe secundă.

Teorie Structurată

7.1 — Input Assembler

GPU primește vertex buffer și index buffer. Definește triunghiuri din vertici.

Index buffer reduce redundanța: un vertex poate fi folosit în mai multe triunghiuri.

7.2 — Vertex Shader

Execută transformări: model → world → view → projection.

Calculează poziții și trimite atribute (normale, UV) către rasterizer.

7.3 — Rasterization

Transformă triunghiuri în pixeli (fragmente). Determină ce pixeli sunt acoperiți.

Fiecare fragment primește interpolare de atribute (culoare, UV).

7.4 — Fragment/Pixel Shader

Calculează culoarea finală. Texturi, lumină, umbre, reflexii.

Pixel shader este de obicei cea mai grea etapă (milioane de pixeli).

7.5 — Post-processing și Output

Etape finale: tone mapping, blur, bloom, anti-aliasing.

Rezultatul este scris în framebuffer și afișat pe ecran.

Legătura Fizică – Informatică

Fizică: GPU execută mii de shader threads în paralel.

Informatică: Fiecare etapă adaugă latență. Optimizezi cea mai lentă etapă.

Aplicare Directă

Exemplu: Frame 4K 60 FPS

4K = 8.3M pixeli. 60 FPS = 498M pixeli/sec. Pixel shader trebuie să fie extrem de eficient.

Dacă pixel shader durează 300 instrucțiuni, GPU trebuie să execute ~150 miliarde instrucțiuni/sec.

Exemplu Real: Bottleneck Pixel Shader

Un joc folosește efecte de iluminare costisitoare. Pixel shader devine bottleneck, FPS scade de la 60 la 35.

Optimizare: reduce sampling texturi și folosește bake lighting. FPS revine la 60.

Probleme Frecvente

⚠️ Overdraw

Randarea acelorași pixeli de mai multe ori (transparențe). Crește costul fragment shader.

⚠️ Shaders prea complicați

Instrucțiuni multe per pixel. GPU saturat, FPS scade.

Recapitulare

  • Pipeline: input → vertex → raster → pixel → post-process.
  • Pixel shader este de obicei bottleneck-ul principal.
  • 4K crește de 4× numărul de pixeli față de 1080p.
  • Optimizările vizează etapa cea mai lentă.

Quiz — 5 Întrebări

Întrebarea 1

Etapa care transformă triunghiuri în pixeli:

  • a) Vertex shader
  • b) Rasterization
  • c) Post-processing
  • d) Input assembler
Arată răspunsul

b) Rasterization

Întrebarea 2

Pixel shader calculează:

  • a) Poziția vertexului
  • b) Culoarea finală a pixelului
  • c) Index buffer
  • d) Layout de memorie
Arată răspunsul

b) Culoarea finală

Întrebarea 3

4K are de câte ori mai mulți pixeli decât 1080p?

  • a) 2×
  • b) 3×
  • c) 4×
  • d) 8×
Arată răspunsul

c) 4×

Întrebarea 4

Overdraw înseamnă:

  • a) Pixeli calculați o singură dată
  • b) Pixeli randati de mai multe ori
  • c) Texturi mai mici
  • d) GPU idle
Arată răspunsul

b) Randare multiplă

Întrebarea 5

Etapa finală înainte de display:

  • a) Vertex shader
  • b) Input assembler
  • c) Post-processing
  • d) RAM training
Arată răspunsul

c) Post-processing

Exercițiu Aplicat

🧠 Exercițiu: Cost Pixel Shader

Cerință: 1080p 60 FPS = 124M pixeli/sec. Pixel shader 200 instrucțiuni/pixel. Câte instrucțiuni/sec?

Arată rezolvarea

124M × 200 = 24.8 miliarde instrucțiuni/sec.
