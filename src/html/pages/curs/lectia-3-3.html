Modul 3 — Logică Digitală

Lecția 3.3 — Flip-flop și Registre

Cum computerele memorează? Circuite cu feedback care țin biți "țintuiți".

Începe Lecția

Introducere

Flip-flop: Circuit cu 2 stări stabile. Ține un bit memorat (0 sau 1) până cand Signal de set/reset îl schimbă.

Registru: Grup de flip-flop-uri. Exemplu: 32 flip-flop-uri în paralel = registru 32-bit care ține o valoare de 32 biți.

Fără flip-flop, computerul nu ar putea ține valori între operații. Ar uita totul la ciclu următor.

Teorie Structurată

3.1 — SR Flip-flop (Set-Reset)

Intrări: S (Set) și R (Reset).

Comportament: S=1 → Q=1 (memorat). R=1 → Q=0. Daca ambii 0, stare anterioară ținută. S=R=1 = undefined.

Implementare: 2 porți NOR cu feedback. Ieșirea unui NOR intră în celălalt. Feedback-ul menține starea.

3.2 — D Flip-flop (Data)

Intrări: D (data) și CLK (clock).

Comportament: La fiecare flanc de clock (0→1), valoarea D se captează și memorează în Q.

Aplicație: Cel mai des folosit. Preia și ține daata sincrona cu clock.

3.3 — JK Flip-flop (Jack-Killer)

Intrări: J, K, CLK.

Comportament: J=1 K=0 → Set Q=1. J=0 K=1 → Reset Q=0. J=K=1 → toggle (comută starea). J=K=0 → fără schimbare.

Avantaj: Versiune "imbunătățit" a SR, fără stat nedefinit.

3.4 — Registre 32-bit și Cache

Registru CPU: 32 flip-flop-uri în paralel. Fiecare bit ține o valoare. La clock tick, toți biții se actualizează simultan.

Exemplu PS5: EAX registru (32-bit) ține rezultatul ultimei operații. Next instruction poate citi din EAX.

Cache: Mii de registre mici în silicon = SRAM (Static RAM). Ultra-rapid, fierbinte, scump în curent.

3.5 — Timing și Setup/Hold Time

Setup time: D trebuie stabil ÎNAINTE de flanc clock (ex. 0.5 ns înainte).

Hold time: D trebuie stabil DUPĂ flanc (ex. 0.3 ns după).

Violație timing: Daca D se schimbă în fereastra setup/hold, flip-flop nimeresc stare metastabila → output oscilează → eroare.

Legătura Fizică – Informatică

Fizică: Feedback-ul în porți NOR creeaza stare stabila care nu se schimbă spontan.

Informatică: Acea stare stabilă = memorie pentru 1 bit. Miliarde de biți stocați în CPU/cache.

Aplicare Directă

Exemplu: Program Counter

Registru special = Program Counter (PC). Ține adresa instrukciunii curente (ex. 0x80000000).

Fiecare instrucție: incrementeaza PC cu 4 (32-bit ARM = 4-byte instrucțiuni). Flip-flop-uri iau valoarea noua, o memoreaza.

Următoarea instrucțiune: citește din PC, executa, apoi incrementează din nou. Proces repetat 2 GHz pe PS5.

Exemplu Real: Cache Hit/Miss Detectare

Scenario: CPU cere data de la adresa 0x1000. Cache controller compară adresa cu 64 D flip-flop-uri (tag bits).

Daca match (cache hit): Flip-flop-uri memorează valid bit = 1. Data se serveste din cache (~3 ciclu CPU).

Daca nu match (miss): Fetch din RAM (~200 ciclu). Flip-flop-uri se actualizeaza cu noul tag și valid=1.

Probleme Frecvente

⚠️ De ce timing violations sunt grave?

Flip-flop în metastabilitate = ieșire neobservabil. Poate lua orice valoare. Propagă erori în calcule ulterioare. Debug = imposibil.

⚠️ De ce cache mai rapid decât RAM?

Cache = SRAM (flip-flop-uri) = milioane per milimetru pătrat. RAM = DRAM (capacitori) = trebuie refresh periodic. Latență SRAM 3ns vs DRAM 50ns+.

Recapitulare

  • Flip-flop: Circuit cu 2 stări stabile. Ține 1 bit memorat.
  • SR: Set/Reset, cu stare nedefinită.
  • D: Data, sincrona cu clock. Cel mai comun.
  • JK: Improved, cu toggle capability.
  • Registru: Mii de flip-flop = cache, program counter, CPU flags.
  • Setup & hold time violații = metastabilitate = bugs imposibili de reprodus.

Quiz — 5 Întrebări

Întrebarea 1

Flip-flop cu feedback ține información datorită:

  • a) Condensatorilor mari
  • b) Circuitului cu feedback în stare stabila
  • c) Tensiunii VCC constant
  • d) Clock-ului regulat
Arată răspunsul

b) Circuitul cu feedback în stare stabila

Întrebarea 2

Ce face D flip-flop?

  • a) Memorează valoarea D la flanc clock
  • b) Comută alternativ 0-1
  • c) Set/Reset după logică
  • d) Stochează cu intârziere aleatoare
Arată răspunsul

a) Memorează valoarea D la flanc clock

Întrebarea 3

32 flip-flop-uri în paralel se numesc:

  • a) Cache line
  • b) Registru 32-bit
  • c) Memorie RAM
  • d) Buffer serial
Arată răspunsul

b) Registru 32-bit

Întrebarea 4

JK flip-flop cu J=1 K=1 face:

  • a) Set Q=1
  • b) Reset Q=0
  • c) Toggle (schimbă starea)
  • d) Nimic (hold)
Arată răspunsul

c) Toggle (comută starea)

Întrebarea 5

Timing violation = metastabilitate. Ce se întâmplă?

  • a) Cicle pierdute
  • b) Flip-flop ia stare nedefinita, output oscilează, calcule eronate
  • c) Clock se opreste
  • d) Nevoie de reboot
Arată răspunsul

b) Flip-flop în stare nedefinita, oscilație, erori — Very serious!

Exercițiu Aplicat

🧠 Exercițiu: Registru 4-bit SRAM

Cerință: Simulează 4 D flip-flop-uri. Input = 1010 binar, Clock = 1. Memoreaza.

  1. Înainte: Q3 Q2 Q1 Q0 = 0000 (neinițializat)
  2. Setup: D3 D2 D1 D0 = 1010
  3. Clock tick: Flip-flop-uri captează → Q3 Q2 Q1 Q0 = 1010 memorat
  4. Clock inactiv: Data la D se schimbă. Q rămâne 1010.
  5. Ciclu următor: Input = 0101. Clock tick: Q = 0101 (nou memorat)
Arată rezolvarea

Simulare 4-bit D Flip-flop Registry:

  • Initial: Q = 0000 (unknown/reset)
  • Ciclu 1: D = 1010 → Clock → Q = 1010 (memorat)
  • Ciclu 2: D = 0101 → Clock → Q = 0101 (memorat)
  • Ciclu 3: D = 1111 → Clock → Q = 1111 (memorat)

Caracteristica: Orice date D se captureaza și ține până ciclu următor. Sincron cu clock.

Timp răspuns: ~1-2 ns (propagation delay prin 4 flip-flop).
