Modul 4 — Arhitectură Hardware

Lecția 4.4 — Bus și Interconectări

Magistralele care mută datele între CPU, GPU, RAM, SSD și I/O. Fără bus, totul e izolat.

Începe Lecția

Introducere

Bus: Canal de comunicare (linii fizice + protocol). Transfer date și control între componente.

PS5 are bus intern între CPU, GPU, RAM (IF bus), plus PCIe 4.0 pentru SSD, USB pentru periferice.

Bandwidth bus determină cât de repede pot circula datele. Un CPU rapid fără bus = blocaj.

Teorie Structurată

4.1 — Bus Tipuri

Address bus: Transportă adresă memorie (32-bit, 64-bit).

Data bus: Transportă date (64-bit, 128-bit). Mai lat = mai multe date/ciclu.

Control bus: Semnale control: read/write, clock, interrupt, reset.

4.2 — Bandwidth și Latență

Bandwidth: Cantitate de date per sec. = (bus width) × (clock frequency).

Exemplu: 128-bit bus la 1 GHz = 128/8 bytes = 16 bytes/ciclu → 16 GB/s.

Latență: Timp până datele ajung. Un bus poate avea bandwidth mare dar latență mare (ex. PCIe).

4.3 — Bus în PS5

Memory bus: CPU/GPU la GDDR6. 256-bit bus @ 14 Gbps → 448 GB/s.

PCIe 4.0: SSD intern. 16 lane × 2 GB/s = 32 GB/s raw. PS5 real = 5.5 GB/s (controller limit).

USB 3.1: 10 Gbps = 1.25 GB/s, pentru externe.

4.4 — Interconnect Topology

Shared bus: Toate modulele partajează aceeași magistrală. Pro: simplu. Con: coliziune data. Scalare slabă.

Crossbar: Switch care conectează orice cu orice. Latență mică, scump.

Ring: Pachetele circulă în cerc. Intel ring bus. Latență medie.

Mesh: Grid de interconnect. AMD Infinity Fabric (PS5). Scalare bună.

4.5 — Bus Contention și QoS

Contenție: CPU și GPU cer simultan acces la RAM. Se formează coadă. Latență crește.

QoS (Quality of Service): Bus controller prioritizează anumite fluxuri. De exemplu, audio buffer prioritate pentru a evita drop-out.

Real World: Streaming textures + audio + physics = bus overloaded. QoS controlează frame pacing.

Legătura Fizică – Informatică

Fizică: Bus = linii de cupru pe PCB. Semnalele sunt impulsuri electrice. Limitate de rezistență, capacitate, inductanță.

Informatică: Protocoale (PCIe, AXI, GDDR). Control starea tranzițiilor și integritatea datelor.

Aplicare Directă

Exemplu: GPU vs CPU Contention

GPU cere 300 GB/s bandwidth pentru frame buffer + textures. CPU cere 30 GB/s pentru physics + AI.

Memory bus = 448 GB/s max. Dacă ambele cer simultan, total 330 GB/s → OK. Dacă GPU spike 450 GB/s → CPU starve.

QoS are rol: menține 30 GB/s pentru CPU pentru a preveni stutter.

Exemplu Real: SSD Streaming PS5

Scenario: Game open-world. SSD stream 5.5 GB/s. Bus PCIe 4.0. Data trece prin decompressor hardware.

Pipeline: SSD → PCIe → IO controller → Decompressor → RAM → GPU. Latența end-to-end ~20 ms.

Bus bottleneck: Dacă PCIe saturat de 6 GB/s și alt device cere 1 GB/s, contension reduce streaming speed. Frame stutter.

Probleme Frecvente

⚠️ De ce bus bandwidth poate limita FPS?

GPU output + texture fetch + CPU data = total > bus bandwidth. Pentru fiecare frame, datele nu pot ajunge la timp. FPS scade.

⚠️ De ce PCIe nu atinge mereu bandwith?

Protocol overhead, handshaking, queue latency. 32 GB/s raw devine 5.5-7 GB/s real (PS5), plus latency penalty.

Recapitulare

  • Bus: Address, data, control. Transferă info între module.
  • Bandwidth: Bus width × frequency. 256-bit @ 14 Gbps → 448 GB/s (PS5).
  • Latență: Timp până data ajunge. Bus poate fi mare dar lent.
  • Topologie: Shared, crossbar, ring, mesh. PS5 folosește Infinity Fabric (mesh).
  • QoS: Prioritizare pentru a evita dropouts (audio, input).
  • Bus contension = FPS drops și stutter. Limitare majoră în consola modernă.

Quiz — 5 Întrebări

Întrebarea 1

Data bus width de 128-bit la 1 GHz → bandwidth?

  • a) 1 GB/s
  • b) 8 GB/s
  • c) 16 GB/s
  • d) 128 GB/s
Arată răspunsul

c) 16 GB/s — 128/8 = 16 bytes per ciclu.

Întrebarea 2

PS5 memory bus?

  • a) 128-bit 7 Gbps
  • b) 256-bit 14 Gbps
  • c) 512-bit 7 Gbps
  • d) 128-bit 14 Gbps
Arată răspunsul

b) 256-bit 14 Gbps — 448 GB/s.

Întrebarea 3

PCIe 4.0 raw bandwidth per 16 lane?

  • a) 2 GB/s
  • b) 8 GB/s
  • c) 16 GB/s
  • d) 32 GB/s
Arată răspunsul

d) 32 GB/s — 16 lanes × 2 GB/s.

Întrebarea 4

QoS în bus înseamnă:

  • a) Overclock bus
  • b) Prioritizează fluxuri critice
  • c) Reduce width bus
  • d) Adaugă mai mult RAM
Arată răspunsul

b) Prioritizează fluxuri critice

Întrebarea 5

Topologia mesh oferă:

  • a) Scalare slabă
  • b) Scalare bună și redundanță
  • c) Latență constantă mereu 1ns
  • d) Zero cost
Arată răspunsul

b) Scalare bună — Mesh crește puncte, dar costă silicon.

Exercițiu Aplicat

🧠 Exercițiu: Bandwidth Budgeting

Cerință: GPU cere 320 GB/s, CPU cere 40 GB/s, SSD streaming cere 10 GB/s. Memory bus PS5 448 GB/s.

  1. Calculați total demand
  2. Este bus saturat?
  3. Dacă GPU spike la 400 GB/s, ce se întâmplă?
  4. QoS decide să rezerva 40 GB/s pentru CPU. Ce rămâne pentru GPU și SSD?
Arată rezolvarea

1. Total demand: 320 + 40 + 10 = 370 GB/s.

2. Bus saturat? 370 < 448. Nu, există headroom 78 GB/s.

3. GPU spike la 400 GB/s: Total = 400 + 40 + 10 = 450 GB/s > 448. Bus saturat. Latency crește, stutter.

4. QoS rezervă CPU 40 GB/s: Rămâne 448-40 = 408 GB/s. GPU + SSD împart 408. Dacă GPU cere 400 și SSD 10, total 410. GPU redus cu 2 GB/s sau SSD throttled.
