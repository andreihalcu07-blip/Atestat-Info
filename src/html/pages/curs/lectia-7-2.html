Modul 7 — Performanță și Grafică

Lecția 7.2 — FPS și Bottleneck

Când ești CPU-bound și când GPU-bound? Cum identifici gâtul de sticlă.

Începe Lecția

Introducere

FPS: Cadre pe secundă. Limitat de etapa cea mai lentă din sistem.

Bottleneck: Componenta care limitează performanța (CPU, GPU, RAM, SSD).

În console, CPU și GPU sunt echilibrate, dar jocurile pot schimba balanța.

Teorie Structurată

7.1 — CPU-bound

CPU nu poate genera suficient de rapid draw calls, physics și AI.

Simptome: GPU usage sub 70%, FPS scade când crește numărul de NPC.

Rezolvare: reduce logică CPU, optimizează AI, folosește multithreading.

7.2 — GPU-bound

GPU este saturat de shading și pixel count.

Simptome: GPU usage 95-100%, CPU usage scăzut.

Rezolvare: scade rezoluția, reduce efecte, optimizează shader-ele.

7.3 — Frame Time

FPS = 1 / frame time. Pentru 60 FPS, frame time = 16.67 ms.

Dacă o etapă durează 20 ms, FPS scade la 50.

Profilarea frame time arată exact unde se pierde timp.

7.4 — Draw Calls și API Overhead

Fiecare draw call are cost CPU. Dacă sunt prea multe, CPU-bound.

Modern APIs (Vulkan, DX12) reduc overhead, dar necesită optimizare de engine.

7.5 — Balanced Pipeline

Ideal: CPU și GPU au frame time similar. Nicio componentă nu stă idle.

În practică, se țintește GPU-bound ușor, pentru a folosi GPU la maxim.

Legătura Fizică – Informatică

Fizică: CPU și GPU au limite de putere și temperatură.

Informatică: Profiling identifică bottleneck-ul și ghidează optimizările.

Aplicare Directă

Exemplu: CPU-bound în jocuri strategice

Joc RTS cu 5,000 unități. CPU calculează AI și pathfinding. GPU stă la 50%.

Reducerea AI update rate sau batching logică crește FPS semnificativ.

Exemplu Real: GPU-bound la 4K

Joc AAA la 4K, ray-tracing activ. GPU 99%, CPU 40%.

Reducerea rezoluției la 1440p crește FPS cu 50% fără impact major vizual.

Probleme Frecvente

⚠️ Interpretare greșită a FPS

FPS mic nu înseamnă neapărat GPU slab. Poate fi CPU-bound sau memory bound.

⚠️ Frame time spikes

Stutter periodic indică asset streaming sau garbage collection, nu neapărat GPU.

Recapitulare

  • CPU-bound: GPU usage mic, CPU saturat.
  • GPU-bound: GPU usage mare, CPU relaxat.
  • Frame time este metricul real pentru FPS.
  • Profiling = cheia optimizării.

Quiz — 5 Întrebări

Întrebarea 1

GPU usage 40% și FPS mic indică:

  • a) GPU-bound
  • b) CPU-bound
  • c) SSD-bound
  • d) RAM-only
Arată răspunsul

b) CPU-bound

Întrebarea 2

Frame time pentru 60 FPS:

  • a) 8.3 ms
  • b) 16.67 ms
  • c) 33 ms
  • d) 5 ms
Arată răspunsul

b) 16.67 ms

Întrebarea 3

Reducerea rezoluției ajută când:

  • a) CPU-bound
  • b) GPU-bound
  • c) PSU-bound
  • d) Update-bound
Arată răspunsul

b) GPU-bound

Întrebarea 4

Draw calls multe afectează:

  • a) GPU only
  • b) CPU overhead
  • c) RAM only
  • d) SSD
Arată răspunsul

b) CPU overhead

Întrebarea 5

Bottleneck =

  • a) Componenta cea mai rapidă
  • b) Componenta cea mai lentă
  • c) Orice componentă
  • d) RAM only
Arată răspunsul

b) Componenta cea mai lentă

Exercițiu Aplicat

🧠 Exercițiu: Identificare Bottleneck

Cerință: GPU 99%, CPU 35%, FPS 30. Ce faci pentru 60 FPS?

Arată rezolvarea

Este GPU-bound. Reduci rezoluția, calitatea shadow/lighting, sau optimizezi shader-ele.
