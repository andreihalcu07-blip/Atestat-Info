Modul 3 — Logică Digitală

Lecția 3.1 — Bit și Sistem Binar

De la volt & masă la 0 și 1. Cum computerele gândesc doar cu binare.

Începe Lecția

Introducere

Bit = BInary digiT. 0 sau 1. Nimic altceva. Microelectronica unui procesor se reduce la: acesta e mai mult decât 0.5V (1) sau mai puțin (0)?

System binar: Folosește 2 cifre (0 și 1) versus sistemul zecimal care folosește 10 (0-9).

De ce binar? Pentru că volt și masă sunt stări stabile. Easy to implement, hard to corrupt. Orice PS5 crede în binar.

Teorie Structurată

3.1 — Ce e Bit?

Bit fizic: Un MOSFET cu dren conectat la VCC (3.3V) și sursă la masă (0V).

Dacă gate e 0V: MOSFET OFF → dren "pluteşte" la VCC prin pull-up rezistor → ieșire = 1

Dacă gate e 3.3V: MOSFET ON → dren = 0V → ieșire = 0

Deci: Bit = (V_node > 1.65V) ? 1 : 0. Orice mai sus de jumătate se crede 1. Mai jos = 0.

3.2 — Sistem Binar

Zecimal: 467 = 4×10² + 6×10¹ + 7×10⁰ = 400 + 60 + 7.

Binar: 01110110 = 0×2⁷ + 1×2⁶ + 1×2⁵ + 1×2⁴ + 0×2³ + 1×2² + 1×2¹ + 0×2⁰ = 64 + 32 + 16 + 4 + 2 = 118 zecimal.

Conversii rapide:

  • 2⁰ = 1, 2¹ = 2, 2² = 4, 2³ = 8, 2⁴ = 16, 2⁵ = 32, 2⁶ = 64, 2⁷ = 128, 2⁸ = 256
  • 255 zecimal = 11111111 binar (8 biți max) = 2⁸ - 1
  • 1024 zecimal = 10000000000 binar (11 biți)

3.3 — Byte, Cuvânt, Megabyte

Byte: 8 biți. Unitate de bază în memoriei și procesare. Interval: 0-255 zecimal (00000000-11111111 binar).

Word: De obicei 32 biți (4 bytes) pe consolele moderne. PS5 CPU = 64-bit (8 bytes). Interval: 0-4,294,967,295 (2³² - 1).

Megabyte (MB): 1,048,576 bytes (2²⁰). PS5 RAM = 16 GB = 16 × 1024 MB = 16,777,216 KB.

Memorie adrese: 16 GB = 2³⁴ bytes. CPU poate adresa din 0 la (2³⁴ - 1).

3.4 — Operații Binare Clasice

AND: 1 AND 1 = 1. Altfel 0. Mascare.

OR: 0 OR 0 = 0. Altfel 1. Setare.

XOR: Egal = 0, diferit = 1. Comparare logică.

NOT: 0→1, 1→0. Inversare.

Exemplu: 10101010 AND 11110000 = 10100000 (pastrează doar biții unde amândoi sînt 1).

3.5 — Reprezentare Numere Negative

Unsigned: Toate biții sunt valori pozitive. 8 biți = 0-255.

Signed (Two's Complement): Bitul MSB (most significant bit) e semn. 8 biți signed = -128 la +127.

Exemplu: 01111111 = +127. 10000000 = -128. -1 = 11111111 (binar).

Two's Complement formula: Pentru negare, inversează toți biții și adaugă 1. De ex, 5 (00000101) → inversat (11111010) → plus 1 (11111011) = -5.

Legătura Fizică – Informatică

Fizică: Volt & masă sunt stări electrice stabile în silicon.

Informatică: Sunt traduse în 0 și 1, apoi în numere, text, culori, sunet.

Aplicare Directă

Exemplu: Stocare Culori in PS5

Pixel roșu: 11111111 (255 roșu) 00000000 (0 verde) 00000000 (0 albastru) = FF0000 hex.

3 bytes per pixel (RGB). Ecran 1080p = 1920×1080 pixeli = 2,073,600 pixeli × 3 bytes = 6.2 MB pe frame.

La 60 FPS: 6.2 MB × 60 = 372 MB/sec de bandwidth memorie doar pentru imagini. GPU-ul PS5 gestionează asta.

Exemplu Real: Adrese Memorie PS5

PS5 16 GB RAM. Adresar 64-bit = 2⁶⁴ posibilități potențiale (but only 16 GB mapped).

Address 0x0000000000 = primii biți din RAM.

Address 0x3FFFFFFFF = ultimii biți (16GB - 1 byte).

Variabilă integer la address 0x1000: ocupă bytes 0x1000-0x1003 (4 bytes). Următoarea variabilă la 0x1004.

Probleme Frecvente

⚠️ De ce nu pot stoca -5 în unsigned byte?

Unsigned byte = 0-255. -5 necesită signed byte (-128 la 127). Dacă forțezi, -5 se interpretzează ca 251 în unsigned.

⚠️ De ce 1 + 1 = 10 în binar?

Pentru că după 1 (1 zecimal) vine 10 (2 zecimal). E baza binar, nu zecimal.

Recapitulare

  • Bit: 0 sau 1, stări electrice (masă sau VCC).
  • Byte: 8 biți, 0-255 unsigned.
  • Word: 32-64 biți, unitate intrare procesor.
  • Binar: Putere lui 2 pentru conversii și indexare.
  • Two's Complement: Reprezentare numere negative in binar.
  • Orice se reduce la 0 și 1. Console-ul PS5 are miliarde de biți activați/dezactivați per secundă.

Exercițiu Aplicat

🧠 Exercițiu: Reprezentare Buffer Memorie

Cerință: Scrie în binar și apoi zecimal următoarele:

  1. Culoare roșu pur: 11111111 (255R) 00000000 (0G) 00000000 (0B) — exprimi în bytes distinct
  2. Memoria unei variabile color (3 bytes) la adresa 0x1000 în PS5
  3. Alocă 256 biți pentru o structură. Exprimă în bytes și megabytes dacă ai 1000 structuri identice
Arată rezolvarea

1. Culoare roșu:

  • Byte 1 (R): 11111111 = 255 zecimal
  • Byte 2 (G): 00000000 = 0 zecimal
  • Byte 3 (B): 00000000 = 0 zecimal
  • Culoare = FF0000 hexazecimal

2. Memorie structură:

  • Adresa 0x1000 = byte 1 (roșu = 255)
  • Adresa 0x1001 = byte 2 (verde = 0)
  • Adresa 0x1002 = byte 3 (albastru = 0)

3. Alocare 256 biți:

  • 256 biți = 32 bytes per structură
  • 1000 structuri × 32 bytes = 32,000 bytes = 31.25 KB ≈ 0.03 MB
